# NNN GAMES 홈페이지 운영 중심 개발 계획

작성일: 2026-03-06
개정: 3차

## 1. 제품 목적 재정의
- 홈페이지의 1차 목적은 회사 소개보다 `자체 운영 중인 프로젝트의 지표와 관련 자료를 보기 쉽게 관리`하는 것이다.
- 따라서 홈 화면의 정보 우선순위는 아래 순서를 따른다.
1. 운영 프로젝트 핵심 KPI
2. 프로젝트별 자료 링크(플레이, 트레일러, 그룹, 기사 등)
3. 지표 갱신 시점과 운영 상태
4. 보조 신호로서의 커뮤니티 규모
5. 회사 일반 소개 정보

## 2. 이번 개정에서 제거한 요소
- `Studio Snapshot` 패널 제거
- 히어로 내 문의 CTA 제거
- 히어로 하단 제작 방식/강점 섹션 제거

## 5. 화면 원칙
- 홈은 `소개 페이지`이면서 동시에 운영 프로젝트 현황을 빠르게 확인하는 페이지로 취급한다.
- 새로운 섹션을 추가할 때는 “이 정보가 운영 판단에 직접 도움이 되는가”를 먼저 검증한다.
- 프로젝트 카드의 정보 밀도는 높이되, 첫 스캔에서 읽히도록 KPI와 링크를 묶어서 배치한다.
- 커뮤니티 섹션은 홈 상단 흐름에 유지하되, 프로젝트 지표 섹션과 역할이 겹치지 않게 분리한다.

## 6. 검증 항목
- `cmd /c npm run build:css`
- `cmd /c npm run update:metrics`
- 홈 히어로에 불필요한 부가 패널/CTA가 제거되었는지 확인
- 홈 프로젝트 카드에서 KPI, 자료 링크, 갱신일이 노출되는지 확인
- KO/EN/JA 전환 시 히어로/홈 운영 문구가 함께 바뀌는지 확인
- `projects.json`/`communities.json`의 집계 숫자가 각 `status/flag` 기준과 일치하는지 확인

## 7. 이번 개정 결과
- 홈 히어로를 목적 중심 구조로 축소했다.
- 커뮤니티 섹션을 프로젝트 섹션보다 위에 유지하도록 순서를 조정했다.
- 홈 프로젝트 카드에 핵심 지표, 자료 링크, 최근 갱신일을 추가했다.
- 프로젝트 히어로 집계를 `status + reporting flag` 기준으로 명시화했다.
- 커뮤니티 집계를 별도 설정 JSON 기반으로 일원화했다.
- CSS 재빌드와 JS 구문 점검, 실제 메트릭 갱신 실행을 다시 수행했다.
